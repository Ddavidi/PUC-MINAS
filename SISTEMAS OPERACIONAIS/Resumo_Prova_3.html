Sistemas Operacionais — Sistemas de Arquivos, I/O, FAT, File Carving e Esteganografia

1. O Conceito de Arquivo e Sistemas de Arquivos

Um Sistema de Arquivos é a parte do SO responsável por gerenciar a forma como os dados são armazenados e recuperados do disco.

Conceitos Básicos

  • Arquivo: Unidade lógica de armazenamento. Seus atributos (nome, tipo, tamanho, permissões) ficam armazenados em uma estrutura chamada diretório ou i-node.
  • Acesso Sequencial: Dados lidos em ordem, do início ao fim (ex: fitas magnéticas).
  • Acesso Direto (Randômico): Permite pular diretamente a um bloco específico do arquivo sem ler os anteriores (essencial para bancos de dados).

Métodos de Alocação (No Disco)

Como os SOs distribuem os arquivos pelos blocos do disco:

  • CONTÍGUA O arquivo ocupa blocos vizinhos no disco.
    Acesso extremamente rápido (pouco seek da cabeça).
    Sofre de grave fragmentação externa e dificuldade para aumentar o arquivo.
  • ENCADEADA Blocos espalhados, cada um aponta para o próximo.
    Zero fragmentação externa, muito fácil aumentar o arquivo.
    Acesso direto lento (é preciso ler o bloco 1 para achar o 2, etc).
  • INDEXADA Um bloco central (índice) guarda todos os ponteiros.
    Acesso direto excelente, sem fragmentação externa.
    Desperdício (fragmentação interna) em arquivos muito pequenos, pois gasta um bloco inteiro só pro índice.

2. File Allocation Table (FAT)

O FAT System é uma evolução inteligente e essencial da Alocação Encadeada.

  • O SO cria uma tabela (a FAT) no início do volume lógico do disco e a carrega inteiramente para a Memória RAM.
  • Cada entrada na tabela representa um bloco (cluster) do disco. Se um arquivo usa o bloco X, a entrada X na tabela contém o número do próximo bloco.
  • A Grande Vantagem: O acesso randômico fica rápido! O SO não precisa ler o disco mecanicamente para seguir a corrente de ponteiros; ele faz isso na velocidade da memória RAM e depois vai ao disco apenas uma vez, direto no bloco que precisa ler.

3. I/O (Entrada e Saída) e Discos

DMA (Acesso Direto à Memória)

Em vez de a CPU gastar tempo movendo cada byte do disco para a RAM, ela terceiriza o trabalho para a Controladora de DMA. O DMA transfere os dados pelos barramentos e avisa a CPU com uma interrupção apenas quando termina, liberando a CPU para outras tarefas.

Desempenho de Discos Magnéticos

O tempo total para acessar e transferir algo do disco é a soma de:

  • Tempo de Busca (Seek): Mover o braço mecânico para o cilindro/trilha correta (é a etapa mais demorada).
  • Atraso Rotacional (Latência): Esperar o prato girar até que o setor passe embaixo da cabeça de leitura.
  • Tempo de Transferência: O tempo para ler os dados e enviá-los à memória.

    DICA PROVA O Interleaving (entrelaçamento) é usado para dar tempo da controladora transferir um bloco para a RAM antes que o próximo bloco passe sob a cabeça, evitando perder uma rotação inteira do disco.

4. File Carving (Esculpimento)

Técnica avançada de Computação Forense para recuperar arquivos apagados ou de discos formatados, quando a tabela de sistema de arquivos (FAT, MFT) foi destruída.

  • O software varre o disco "cru" setor por setor (raw data).
  • Procura por Magic Numbers (Assinaturas). Todo arquivo JPEG, por exemplo, começa com os bytes FF D8 e termina com FF D9.
  • Ao achar o início e o fim correspondentes, o programa "esculpe" os blocos que estão no meio e recupera a foto, sem depender do Sistema Operacional saber onde o arquivo estava.

5. Esteganografia

Diferente da Criptografia (que transforma a mensagem em código embaralhado visível), a Esteganografia é a arte de ocultar a EXISTÊNCIA da mensagem secreta.

  • Um dado confidencial é embutido dentro de um "arquivo hospedeiro" perfeitamente normal (uma imagem ou áudio inofensivo).
  • Técnica LSB (Least Significant Bit): Substitui o último bit das cores de cada pixel por bits da mensagem secreta. A olho nu, a foto continua idêntica, mas softwares extraem a mensagem perfeitamente. Muito usado em espionagem industrial e vazamento de dados dissimulado.
